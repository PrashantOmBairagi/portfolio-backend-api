# Portfolio Backend API

A Spring Boot REST API powering my personal portfolio website.

This project serves as the backend for handling contact form submissions, persistence using PostgreSQL, and future portfolio analytics such as visitor tracking, resume downloads, and interaction monitoring.

## Features

- Contact form API
- RESTful endpoints
- PostgreSQL database integration
- Spring Data JPA persistence
- Cloud database connectivity (Neon PostgreSQL)
- Deployment-ready architecture

## Tech Stack

- Java
- Spring Boot
- Spring Data JPA
- PostgreSQL
- Maven
- Postman
- Neon Database

## Current Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/contacts` | Submit contact form |
| GET | `/api/contacts` | Get all contact entries |
| GET | `/api/contacts/{id}` | Get contact by ID |
| DELETE | `/api/contacts/{id}` | Delete contact |

This project is part of my backend engineering journey to learn real-world REST APIs, database persistence, deployment workflows, and scalable backend architecture using Java and Spring Boot.

Future plans include:
